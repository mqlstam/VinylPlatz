VinylPlatz - About Page

Introduction

VinylPlatz is a full-stack web application that serves as an online marketplace for buying, selling, and reviewing vinyl records. It provides a user-friendly interface for users to explore and interact with a wide range of vinyl records. The application is developed using Angular for the frontend and NestJS for the backend.

Entities and Relationships

VinylPlatz comprises the following entities:

  1. User: Represents users of the application and contains information for authentication and personal data. Attributes include username, email, password_hash, location, and role.
  2. Record: Represents the vinyl records available on the platform. It includes attributes such as recordID, title, artist, genre, releaseYear, description, and image_url.
  3. Transaction: Tracks the sale or purchase actions of a vinyl record. It captures information such as transactionID, buyer, seller, record, price, transactionDate, and status.
  4. Rating: Stores user ratings and reviews for purchased vinyl records. It contains attributes like reviewID, user, plate, rating, reviewText, and reviewDate.

The relationships between these entities are as follows:

  1. A User can own multiple Records (1:N relationship).
  2. A User can engage in multiple Transactions as a buyer or seller (1:N relationship).
  3. A Record can be involved in multiple Transactions (1:N relationship).
  4. A User can provide multiple Ratings (1:N relationship).
  5. A Record can have multiple Ratings (1:N relationship).

Frontend Functionalities

VinylPlatz provides the following key functionalities in its frontend:

  1. User Registration and Authentication: Users can register and authenticate themselves to access the full features of the application.
  2. Vinyl Record Listings: Users can view a list of available vinyl records, including their titles, artists, genres, and prices.
  3. Record Details: Users can explore detailed information about a specific vinyl record, such as its description, release year, and average rating.
  4. Adding, Editing, and Deleting Records: Authenticated users can add new vinyl records to the marketplace, edit existing records, and delete records they own.
  5. Transactions: Users can initiate and complete transactions for purchasing or selling vinyl records.
  6. Rating and Reviewing: Users can rate and provide reviews for vinyl records they have purchased.
  7. Filtering and Sorting: Users can filter and sort vinyl records based on various criteria, such as genre, artist, and price.
  8. Transaction History: Users can view a summary of their transaction history, including past purchases and sales.
  9. Ratings Overview: Users can see an overview of the ratings and reviews received by a particular vinyl record.

Backend Functionalities

The backend of VinylPlatz, developed using the NestJS framework, provides the following functionalities:

  1. API Endpoints: The backend exposes API endpoints to perform CRUD operations on the entities (User, Record, Transaction, Rating).
  2. Authentication and Authorization: Secure endpoints are implemented to handle user authentication and authorization based on user roles.
  3. Transaction Management: The backend handles the logic for processing transactions and updating the status of transactions based on buyer and seller actions.
  4. Rating Calculation: The backend calculates the average rating for each vinyl record based on the ratings provided by users.
  5. Advanced Queries: Complex queries are implemented to support filtering and sorting of vinyl records, retrieving transaction history, and accessing ratings.

Testing and Implementation

Both the frontend and backend applications of VinylPlatz are thoroughly tested using test cases to ensure they function as intended. A CI/CD pipeline is employed to automate the testing and deployment process, ensuring reliable and efficient development practices. The application is deployed on a cloud platform, enabling online access and real-world usage.
