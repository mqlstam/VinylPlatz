VinylPlatz - About Page

Introduction

VinylPlatz is a full-stack web application that serves as an online marketplace for buying, selling, and reviewing vinyl records. It provides a user-friendly interface for users to explore and interact with a wide range of vinyl records. The application is developed using Angular for the frontend and NestJS for the backend combined in a Nx Monorepo Workspace.

Entities and Relationships

VinylPlatz comprises the following entities:

  1. User: Represents users of the application and contains information for authentication and personal data. Attributes include username, email, password, role, firstName, lastName, profileImage, and interestedGenres.
  2. Album: Represents the vinyl records available on the platform. It includes attributes such as title, artist, releaseDate, genre, description, coverImageUrl, and purchasedBy.
  3. Transaction: Tracks the sale or purchase actions of a vinyl record. It captures information such as albumId, buyerId, sellerId, and transactionDate.

The relationships between these entities are as follows:

  1. A User can own multiple Albums (1:N relationship).
  2. A User can engage in multiple Transactions as a buyer or seller (1:N relationship).
  3. An Album can be involved in multiple Transactions (1:N relationship).
  4. A User can have multiple Friends (N:M relationship).

Frontend Functionalities

VinylPlatz provides the following key functionalities in its frontend:

  1. User Registration and Authentication: Users can register and authenticate themselves to access the full features of the application.
  2. Vinyl Record Listings: Users can view a list of available vinyl records, including their titles, artists, genres, and cover images.
  3. Record Details: Users can explore detailed information about a specific vinyl record, such as its description, release date, and genre.
  4. Adding, Editing, and Deleting Records: Authenticated users can add new vinyl records to the marketplace, edit existing records, and delete records they own.
  5. Transactions: Users can initiate and complete transactions for purchasing vinyl records.
  6. Filtering and Sorting: Users can filter and sort vinyl records based on various criteria, such as genre, artist, and release year.
  7. Album Recommendations: Users can view personalized album recommendations based on their interested genres and friend relationships.
  8. Transaction History: Users can view a summary of their transaction history, including past purchases.
  9. Purchased Albums: Users can see a list of albums they have purchased.
  10. Adding Friends: Users can add other users as friends to expand their network.
  11. Interested Genres: Users can specify their interested genres to receive tailored recommendations.

Backend Functionalities

The backend of VinylPlatz, developed using the NestJS framework, provides the following functionalities:

  1. API Endpoints: The backend exposes API endpoints to perform CRUD operations on the entities (User, Album, Transaction).
  2. Authentication and Authorization: Secure endpoints are implemented to handle user authentication using JSON Web Tokens (JWT) and role-based authorization.
  3. Transaction Management: The backend handles the logic for processing transactions and updating the ownership of albums based on buyer and seller actions.
  4. Album Recommendations: The backend generates personalized album recommendations for users based on their interested genres and friend relationships using a graph database (Neo4j).
  5. Error Handling: The backend includes global error handling using interceptors to provide meaningful error responses to the frontend.
  6. Logging: The backend incorporates logging to track important operations and assist in debugging and monitoring.

Testing and Implementation

Both the frontend and backend applications of VinylPlatz are tested using unit tests to ensure the correctness of individual components and services. The project follows a modular architecture using Nx Monorepo Workspace to facilitate code reusability and maintainability. The application leverages MongoDB as the primary database for storing entities and Neo4j as a graph database for generating personalized recommendations.

Conclusion

VinylPlatz aims to provide a comprehensive and user-friendly platform for vinyl record enthusiasts. With features like personalized recommendations, friendship connections, and seamless transactions, VinylPlatz offers a unique and engaging experience for users to explore, purchase, and connect with others who share their passion for vinyl records.
